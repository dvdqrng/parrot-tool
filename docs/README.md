## Documentation Standards

All documentation follows these principles:

- **Clear Structure**: Organized with tables of contents and sections
- **Examples Included**: Real-world examples throughout
- **Searchable**: Clear headings and keywords
- **Up-to-Date**: Updated with code changes
- **Cross-Referenced**: Links between related docs
- **Actionable**: Step-by-step instructions

## How to Read the Docs

### If you're new to Parrot:
1. Start with [README.md](../README.md) in the root directory
2. Follow [Quick Start](../README.md#quick-start) to install
3. Read [Configuration Guide](CONFIGURATION.md) to set up
4. Browse [User Guide](USER_GUIDE.md) to learn features

### If you want to use autopilot:
1. Understand basics in [User Guide](USER_GUIDE.md)
2. Deep dive into [Autopilot Guide](AUTOPILOT.md)
3. Create your first agent
4. Monitor results in activity log

### If you want to contribute code:
1. Read [Contributing Guide](CONTRIBUTING.md)
2. Follow [Development Guide](DEVELOPMENT.md) for setup
3. Review [Architecture Guide](ARCHITECTURE.md) for context
4. Check [API Documentation](API.md) for endpoints

### If you're experiencing issues:
1. Check [Troubleshooting Guide](TROUBLESHOOTING.md) first
2. Search [GitHub Issues](https://github.com/yourusername/beeper-kanban/issues)
3. Ask in Discussions or open new issue

## Document Summaries

### USER_GUIDE.md
Complete walkthrough of using Parrot. Covers basic workflows, message management, drafts, AI features, organization strategies, keyboard shortcuts, tips and tricks, and common workflows. Perfect for new and intermediate users.

### CONFIGURATION.md
Detailed configuration instructions. Covers Beeper integration, AI provider setup (Anthropic, OpenAI, Ollama), writing style training, autopilot configuration, hidden chats, and data management. Essential for initial setup.

### TROUBLESHOOTING.md
Comprehensive troubleshooting guide. Organized by issue type: installation, connections, messages, AI integration, autopilot, performance, Electron app, data storage, and browser-specific issues. Includes solutions and prevention tips.

### AUTOPILOT.md
In-depth guide to the autopilot system. Covers agent creation, human-like behavior simulation, operating modes, goal completion strategies, activity monitoring, technical architecture, best practices, and advanced usage patterns.

### DEVELOPMENT.md
Developer guide for contributors. Covers development setup, code structure, adding features, testing, code style, common tasks, debugging, performance optimization, and deployment. Essential for contributors.

### ARCHITECTURE.md
Technical architecture documentation. Details system architecture, component structure, data flow, state management, API design, autopilot system internals, performance considerations, security architecture, and scalability.

### API.md
Complete API reference. Documents all Beeper API routes, AI API routes, utility routes, authentication, request/response formats, error handling, type definitions, and usage examples. For developers integrating or extending.

### CONTRIBUTING.md
Contribution guide. Covers code of conduct, how to contribute (bugs, features, docs), development setup, coding guidelines, commit conventions, pull request process, testing requirements, and recognition.
